## 🤖 7.2 Redis TTL + namespaces

**Что сделать в коде (AI):**
- Все ключи писать с префиксом проекта/среды (например `app:prod:session:`, `app:prod:cache:`), чтобы не пересекаться с другими приложениями и средами.
- Для каждого ключа задавать TTL (EXPIRE или параметр при SET), где возможно, чтобы ключи не копились бесконечно (сессии, кеш, rate limit счётчики — с разумным TTL).

**Проверка:** В Upstash в списке ключей виден префикс; ключи имеют TTL и не копятся без ограничения.

---

## 🤖 7.3 No PII/secrets in Redis

**Что сделать в коде (AI):**
- Не хранить в Redis пароли, токены, полные email/телефоны/адреса. Сессии — только идентификатор или непрозрачный токен; детали пользователя при необходимости подгружать из БД.
- Сделать код-ревью и/или правило в команде: перед записью в Redis проверять, что значение не содержит PII и секретов.

**Проверка:** Аудит кода и содержимого ключей в Redis: нет email/phone/token в значениях.

## 👤 7.1 Redis TLS

**Куда зайти → что сделать:**
- **Upstash:** В панели проекта открыть **REST API** или **Connect** и скопировать URL. Убедиться, что используется URL с протоколом **`rediss://`** (с двумя «s»), а не `redis://`.
- **Vercel / env:** В переменных окружения (например `UPSTASH_REDIS_REST_URL` или `REDIS_URL`) подставить URL с `rediss://`.

**Проверка:** В env переменная содержит `rediss://`; приложение подключается к Upstash по TLS.
