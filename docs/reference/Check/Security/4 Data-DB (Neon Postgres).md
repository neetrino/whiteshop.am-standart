## 🤖 4.2 DB pooling + connection limits

**Что сделать в коде (AI):**
- Использовать pooled connection string Neon (обычно с суффиксом `-pooler` в хосте или отдельный endpoint для pooling). В serverless (Vercel functions) не создавать новое соединение на каждый запрос — использовать кешированный пул или драйвер с pooling.
- В коде/конфиге задать разумные лимиты и таймауты (например max connections на приложение, connection timeout), чтобы не исчерпывать лимит Neon.

**Проверка:** Под нагрузкой нет всплеска «too many connections», соединения переиспользуются.

## 👤 4.1 DB TLS required

**Куда зайти → что сделать:**
- **Neon:** В панели при создании/копировании connection string выбрать вариант с SSL (обычно по умолчанию `sslmode=require` в строке). Скопировать connection string.
- **Vercel:** **Project → Settings → Environment Variables.** Добавить переменную (например `DATABASE_URL`) со значением из Neon для Production/Preview. Убедиться, что в строке есть `sslmode=require` или эквивалент.

**Проверка:** Строка подключения содержит SSL; приложение подключается к БД только по TLS.

---

## 👤 4.3 DB least privilege

**Куда зайти → что сделать:**
- **Neon:** В **SQL Editor** или через консоль создать отдельную роль для приложения (не использовать суперпользователя). Выдать только нужные права: SELECT/INSERT/UPDATE/DELETE на нужные таблицы, при необходимости EXECUTE на функции. Миграции можно выполнять от отдельной роли с правом на DDL.
- Документировать, какая роль для app, какая для migrations.

**Проверка:** Пользователь приложения не может делать DROP, не имеет доступа к чужим объектам и не является superuser.

---

## 👤 4.4 Backups + restore test

**Куда зайти → что сделать:**
- **Neon:** **Project → Backups** (или настройки плана). Убедиться, что включены автоматические бэкапы (минимум daily, если доступно — PITR).
- Один раз выполнить **restore** в отдельный branch или staging: создать копию из бэкапа, подключить приложение к восстановленной БД и убедиться, что данные и приложение работают.

**Проверка:** В настройках Neon бэкапы включены; восстановление из бэкапа успешно проверено.
